## Переписанный вариант

Пока все обсуждают Codex от OpenAI и Claude Code от Anthropic, есть третий игрок — **OpenCode**. И он, пожалуй, самый интересный.
## Что такое OpenCode
OpenCode — это опенсорсный AI-кодинг-агент. Работает в трёх режимах: терминал, десктопное приложение и IDE-расширение. Проект создан компанией [SST](https://sst.dev) (известной по фреймворку для serverless), написан на TypeScript, под MIT-лицензией.
Цифры говорят сами за себя: **150 000+ звёзд** на GitHub, **850+ контрибьюторов**, **11 000+ коммитов**, **6.5 миллионов** активных пользователей в месяц. Проекту всего год с небольшим — первый коммит в апреле 2025.
## Чем отличается от конкурентов
Главное отличие OpenCode от Codex и Claude Code — **свобода выбора моделей без привязки к одному вендору**.
OpenCode предлагает два варианта доступа к моделям: **Go** ($10/мес, первый месяц $5) — подписка на проверенные open-source модели (DeepSeek V4, Qwen3.5, Kimi K2.6, GLM-5 и другие). И **Zen** (pay-as-you-go, пополнение от $20) — курированные проприетарные модели с гарантированным качеством.
Но если хочется — можно подключить **любую модель от любого провайдера**: Claude, GPT, Gemini, локальные модели через Ollama. OpenCode не привязан к конкретному вендору, и это одна из его сильных сторон.

## Ключевые фичи
**Мультиинтерфейсность.** Можно работать в терминале, если вы vim-разработчик. Можно поставить десктопное приложение (бета, но уже на macOS/Windows/Linux). Можно воткнуть расширение в VS Code или Cursor.
**Мультиагентность.** OpenCode поддерживает саб-агенты — можно распараллелить задачи, делегировать проверки, организовать конвейер из нескольких агентов.
**Privacy-first.** В отличие от облачных решений, OpenCode не хранит ваш код и контекст. Подходит для чувствительных проектов и NDA.
**Headless-режим Go.**
Это одна из самых сильных фич OpenCode для автоматизации. Go — SDK и CLI-режим, который позволяет использовать OpenCode без интерактивного интерфейса. Вы запускаете его как обычную программу в CI/CD-пайплайне, а он работает как полноценный кодинг-агент.
Как это выглядит на практике:
```bash
opencode run \
  --model claude-sonnet-4 \
  --prompt "Исправь все TypeScript-ошибки в src/ и создай PR"
```
Что происходит: OpenCode берёт ваш код, анализирует, исправляет ошибки — и не задаёт уточняющих вопросов. Работает полностью автономно, как `codex exec` в Codex CLI. Поддерживает все модели, все провайдеры, MCP-серверы для доступа к внешним API.
Зачем это нужно:
- Автоматический рефакторинг в CI при каждом пуше
- Генерация changelog'ов из коммитов
- Проверка типов и линтинг с автоматическими исправлениями
- Миграция легаси-кода по расписанию
По сути — AI-разработчик, который работает пока вы спите.
**ACP-поддержка.** Agent Communication Protocol — открытый протокол для взаимодействия между агентами. Этого нет ни у Codex, ни у Claude Code.
**Plugin SDK.** Можно писать свои расширения, инструменты и интеграции.
## Кто это делает
Проект ведётся командой **SST** (Serverless Stack) — они же **Anomaly**. Это опытная команда с сильным open-source бэкграундом (их фреймворк SST тоже имеет 20K+ звёзд). Судя по скорости роста, ресурсы у проекта серьёзные.
## Сколько стоит
**Сам OpenCode — бесплатный и открытый (MIT).** Подключаете свои API-ключи (OpenAI, Anthropic, Google) — платите провайдерам напрямую.
Встроенные модели OpenCode:
- **Go** — **$10/мес** (первый месяц $5). 14+ open-source моделей: DeepSeek V4 Pro, Qwen3.5 Plus, Kimi K2.6, GLM-5.1, MiniMax M2.7. Щедрые лимиты, без скрытых платежей
- **Zen** — **pay-as-you-go** от $20. Курированные проприетарные модели, протестированные специально для кодинг-агентов. Можно выставить месячный лимит расходов
- **Enterprise** — per-seat. SSO, self-hosting, internal AI gateway. Цена по запросу
## Стоит ли пробовать
Если вы выбираете AI-кодинг-агент и не хотите привязываться к одному вендору — OpenCode выглядит как самый прагматичный выбор. Низкая цена входа, опенсорс, мультиагентность, три интерфейса, ACP, плагины.
Codex силён интеграцией с ChatGPT-экосистемой. Claude Code — качеством reasoning'а на Sonnet/Opus. А OpenCode — свободой выбора и независимостью от вендоров. Каждый найдёт своё.
Главное — конкуренция в этом сегменте накаляется, и это лучшая новость для разработчиков.
